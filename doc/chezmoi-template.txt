1. INTRODUCTION                                 *chezmoi-template-introduction*

chezmoi-template.nvim makes Neovim understand chezmoi (https://chezmoi.io)
source files, so you edit them natively instead of going through
`chezmoi edit`:

- *.tmpl buffers are 'filetype' gotmpl with the deployed target language
  treesitter-injected (a dot_zshrc.tmpl highlights as Go template + zsh).
- A conform.nvim formatter formats templates with the TARGET filetype's
  formatter by masking Go-template spans and restoring them afterwards.
- mini.icons lookups resolve source names (private_dot_config/...) to their
  deployed targets.
- Optional transparent decrypt/encrypt of chezmoi-managed encrypted files
  (*.age, *.asc).
- Template errors surface as |vim.diagnostic| entries on write.
- User commands for apply/diff/preview/source-jumping.

Everything degrades gracefully when the `chezmoi` binary is missing.

2. SETUP                                               *chezmoi-template-setup*

The plugin bootstraps itself with defaults; a setup() call is only needed to
change options. With lazy.nvim: >lua

  {
    "dpezto/chezmoi-template.nvim",
    lazy = false, -- keep false; the plugin lazy-loads itself internally
    opts = {},
  }
<
`lazy = false`, but startup stays cheap: only filetype detection and light
triggers register up front; the heavy setup() is deferred until the first
template opens or a :Chezmoi* command runs. Don't set ft/cmd — the internal
bootstrap handles lazy-loading, and its filetype registration must run at
startup.

Without a plugin spec, options can also be set via >lua

  vim.g.chezmoi_template = { encryption = { enabled = true } }
<
before the plugin loads.

3. OPTIONS                                           *chezmoi-template-options*

Defaults: >lua

  require("chezmoi-template").setup({
    source_dir = nil,            -- nil = auto via `chezmoi source-path`
    inject = { enabled = true }, -- treesitter target-language injection
    format = {
      enabled = true,            -- conform formatter registration
      indent_directives = true,  -- depth-pad column-0 {{- directive blocks
    },
    icons = { enabled = true },  -- mini.icons resolution
    apply = {
      on_save = true,            -- chezmoi apply <target> after writing
      notify = true,             -- notify on successful applies
    },
    notify_on_open = false,      -- notify when opening a managed source file
    redirect = false,            -- deployed file open -> jump to source
    diagnostics = { enabled = true },
    completion = {
      -- hide values of matching data keys in completion docs
      mask = { "secret", "token", "passw", "key", "api" },
    },
    picker = nil,                -- "snacks"|"telescope"|"fzf-lua"|"mini"|
                                 -- "select"; nil = auto-detect
    encryption = {
      enabled = false,           -- opt-in; delegates to chezmoi decrypt/encrypt
      exclude = {},              -- lua patterns of *.age paths to skip
    },
  })
<
                                            *chezmoi-template-indent-directives*
format.indent_directives rewrites the interior padding of column-0 `{{-`
whole-line directives so template nesting depth reads as indentation
(1 space + 2 per level): >

  {{- range $name, $spec := .packages.apps }}
  {{-   $roles := get $spec "role" }}
  {{-   if $ok }}
  {{-     $via := get $spec "via" }}
  {{-   end }}
  {{- end }}
<
Directives with leading indentation (embedded in code layout) keep their
single space.

                                                  *chezmoi-template-encryption*
encryption (opt-in) transparently decrypts managed *.age / *.asc files on read
and re-encrypts on write via `chezmoi decrypt` / `chezmoi encrypt`: identities,
recipients, tool choice (age/rage/builtin, even gpg) come from chezmoi's own
encryption config. Decrypted content never touches disk (no swap/undo file).
encryption.exclude lua patterns leave matching paths as plain binary.

4. COMMANDS                                         *chezmoi-template-commands*

                                                                *:ChezmoiApply*
:ChezmoiApply         Apply the current buffer's deploy target.
:ChezmoiApply!        Apply all targets.

                                                                 *:ChezmoiDiff*
:ChezmoiDiff          Diff for the current buffer's target in a split; the
                      whole state when the buffer is not chezmoi-managed.
                      `q` closes the split.

                                                               *:ChezmoiTarget*
:ChezmoiTarget        Echo the current buffer's deploy target path.
:ChezmoiTarget!       Open the deployed file.

                                                               *:ChezmoiSource*
:ChezmoiSource        From a deployed managed file, jump to its chezmoi
                      source. (Set redirect = true to do this automatically.)

                                                              *:ChezmoiPreview*
:ChezmoiPreview       Toggle a vertical split with the rendered template
                      (`chezmoi execute-template`), typed as the target
                      filetype and re-rendered on every write. The preview
                      buffer is named chezmoi-preview://<target> (a
                      dot_zshrc.tmpl previews as .zshrc); `q` (or
                      running the command again) closes it. Refuses
                      non-template buffers. NOTE: templates calling secret
                      managers render real secrets here.

                                                                 *:ChezmoiPick*
:ChezmoiPick          File picker over the source directory. Backend from
                      the picker option; auto-detects loaded pickers
                      (snacks > telescope > fzf-lua > mini.pick), falling
                      back to |vim.ui.select()|. Lazy-loaded pickers may not
                      be detected — set the option explicitly.

5. COMPLETION                                     *chezmoi-template-completion*

A blink.cmp source for gotmpl buffers (blink.cmp ≥ 0.13 — per-item kind
icons/highlights). Inside {{ … }} it completes template
data keys (from `chezmoi data`; kind icons reflect each value's type, docs
preview the value), template/sprig/chezmoi functions, and Go template
keywords. Outside actions it completes block snippets (if, ifelse, range,
rangekv, with, define, block, comment) expanding to full
`{{- … }}…{{- end }}` pairs. Register it in blink.cmp opts: >lua

  sources = {
    default = { "chezmoi", "lsp", "path", "buffer" },
    providers = {
      chezmoi = { name = "chezmoi", module = "chezmoi-template.blink" },
    },
  }
<
